# Repository Guidelines

## Project Structure
- `src/`: Modular TypeScript source for the userscript runtime.
- `src/index.ts`: Bootstrap entrypoint; injects styles, builds UI, and starts the app.
- `src/app.ts`: Main filtering, routing, storage, UI state, and event behavior.
- `src/styles.ts` and `src/icon.ts`: Embedded CSS and rabbit SVG assets used by the bundle.
- `userscript.metadata.cjs`: Userscript metadata header inserted by Webpack.
- `WTR Lab Delulu Destroyer.user.js`: Tracked distributable generated by `npm run build`.
- `scripts/validate-userscript.cjs`: Metadata validator used by `npm run validate` and CI.

## Development Commands
- `npm install`: Install local dependencies for development.
- `npm ci`: Install dependencies exactly from `package-lock.json`; used by GitHub Actions.
- `npm run build`: Bundle `src/index.ts` with Webpack into `WTR Lab Delulu Destroyer.user.js`.
- `npm run typecheck`: Run strict TypeScript checking without emitting files.
- `npm run validate`: Run typecheck, rebuild the userscript, check generated JavaScript syntax, and validate metadata.
- Lint command: Absent; no lint script or lint config is defined.
- Test command: Absent; no test script or test files are present.
- Single-test command: Absent; no test runner is configured.
- Development server command: Absent; this is a bundled userscript, not a served web app.

## Coding Conventions
- Use TypeScript under `src/` for behavior changes; rebuild the `.user.js` output instead of hand-editing bundled logic.
- Preserve strict TypeScript settings from `tsconfig.json`: `strict`, `noImplicitReturns`, `noFallthroughCasesInSwitch`, and `noUncheckedIndexedAccess`.
- Use ES2020-compatible browser code and Webpack bundling configured in `webpack.config.cjs`.
- Preserve userscript metadata in `userscript.metadata.cjs`; Webpack injects it with `BannerPlugin`.
- Keep the runtime lightweight and avoid tracking, analytics, or remote telemetry; this is stated in `CONTRIBUTING.md`.
- Keep WTR Lab reading pages undisturbed; exclusions and route checks are part of the userscript behavior.
- Follow `.editorconfig`: LF endings, final newline, UTF-8, 4-space indentation, trim trailing whitespace except Markdown.

## Testing & Verification
- Run `npm run validate` before reporting completion for code or metadata changes.
- Use `npm run typecheck` for a faster TypeScript-only check while iterating.
- Use `npm run build` after changing `src/`, `userscript.metadata.cjs`, or Webpack configuration.
- For user-facing behavior changes, update `CHANGELOG.md` per `CONTRIBUTING.md`.
- Include manual verification notes for affected WTR Lab pages when changing filtering, routing, UI, or manager compatibility.
- Do not claim ScriptCat, Violentmonkey, Stay, Tampermonkey, Firefox, Chrome, or iOS behavior is verified unless it was actually checked.

## Safety & Change Management
- Preserve unrelated user changes; check `git status --short` before editing.
- Use strict Conventional Commits 1.0.0 with no emojis.
- Split unrelated logical concerns into multiple atomic commits by default.
- Use one commit only when the full diff is one logical unit or the user explicitly approves a single mixed-concern commit.
- Do not edit `package-lock.json` unless dependency changes require it.
- Do not edit generated `WTR Lab Delulu Destroyer.user.js` by hand; change source or metadata and run `npm run build`.
- Do not add dependencies, tests, formatters, hooks, CI workflows, or documentation beyond the requested scope.

## Agent Notes
- Read `README.md` first for product scope, supported userscript managers, install links, and Stay/iOS notes.
- Read `CONTRIBUTING.md` before pull-request-style work; it defines local checks and contribution expectations.
- GitHub Actions runs `npm ci` then `npm run validate` on `main` pushes and pull requests.
- The namespace is `https://docs.scriptcat.org/en/`; keep ScriptCat, Violentmonkey, and Stay compatibility in mind when changing metadata or browser APIs.
- Stay compatibility is sensitive to generated code patterns; preserve the existing no-`window.*` generated-bundle behavior unless real-device testing proves otherwise.
